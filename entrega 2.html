1. Diseña los siguientes tipos

Para cada diseño, debe decidir si se usa un record o una clase. Recuerde que los records son adecuados cuando los tipos son inmutables. En el caso de las clases, genere los métodos de Object de forma automática con Eclipse.

Libro

El tipo Libro representa a un libro dentro de una biblioteca. Las propiedades que definen a un libro son:

  • titulo: String que indica el título del libro.
  • autor: String que indica el autor del libro.
  • isbn: String que indica el ISBN del libro. Debe tener 13 caracteres. Todos deben ser numéricos, excepto el último, que puede ser numérico o ‘X’

Este tipo es inmutable.

Además, el tipo Libro debe implementar un método estático llamado parse que reciba una cadena de texto y devuelva un objeto de tipo Libro con los valores correspondientes. El formato del texto es:

titulo, autor, ISBN

Ejemplar

El tipo Ejemplar representa a un ejemplar físico del libro dentro de la biblioteca. Las propiedades que definen a un ejemplar son:

  • libro: Objeto de tipo Libro que indica el libro que representa este ejemplar.
  • codigo: String que indica el código identificativo del ejemplar dentro de la biblioteca.
  • unico: Booleano que indica si el ejemplar es único en la biblioteca. Esta propiedad es modificable

Además, el tipo Ejemplar debe implementar un método estático llamado parse que reciba una cadena de texto y devuelva un objeto de tipo Ejemplar con los valores correspondientes. El formato del texto es :

{titulo, autor, ISBN};codigo;unico

donde unico podrá valer “si” o “no”. En el primer caso, la propiedad deberá valer true, y en el segundo, deberá valer false. Observe bien en el test que cada objeto tenga un valor correcto para esta propiedad.

Usuario

El tipo Usuario representa a una persona que utiliza la biblioteca. Las propiedades que definen a un usuario son:

  • nombre: String con el nombre del usuario.
  • apellidos: String con los apellidos del usuario.
  • dni: String con el DNI del usuario. Debe contener 8 números y una letra

Este tipo es inmutable.

Además, el tipo Usuario debe implementar un método estático llamado parse que reciba una cadena de texto y devuelva un objeto de tipo Usuario con los valores correspondientes. El formato del texto es:

nombre, apellido, dni

Prestamo

El tipo Prestamo representa a un préstamo de un ejemplar a un usuario. Las propiedades que definen a un préstamo son:

  • usuario: Objeto de tipo Usuario que indica el usuario que ha solicitado el préstamo.
  • ejemplar: Objeto de tipo Ejemplar que indica el ejemplar que se ha prestado.
  • fechaSolicitud: LocalDate que indica la fecha en que se ha realizado la solicitud del préstamo.
  • fechaDevolucion: LocalDate que indica la fecha en que se debe devolver el ejemplar. Debe ser posterior o igual a la fecha de solicitud
  • fechaRealDevolucion: LocalDate que indica la fecha en que se ha devuelto el ejemplar. Debe ser posterior o igual a la fecha de solicitud, y puede ser null si aún no se ha devuelto. Esta propiedad es modificable
  • multa: Derivada de tipo Double que indica la multa pendiente de pago por la devolución tardía del ejemplar. Por cada día rebasado, la cuantía será de 1.5€ para los ejemplares que no sean únicos, y de 2.5€ para los que sí lo sean. En el caso de que la devolución se haya realizado a tiempo, su valor será 0. En el caso de que no se haya devuelto el ejemplar, su valor será null.

Además, el tipo Prestamo debe implementar un método estático llamado parse que reciba una cadena de texto y devuelva un objeto de tipo Prestamo con los valores correspondientes. El formato del texto es:

{nombre, apellido, dni}-{{titulo, autor, ISBN};codigo;unico}-fechaSolicitud-fechaDevolucion-fechaRealDevolucion


Por último, no olvide incluir en la representación como cadena el valor de la multa (además del resto de propiedades)
